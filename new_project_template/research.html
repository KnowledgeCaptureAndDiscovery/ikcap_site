Research

The main research thrusts of this work are:

  • Helping users create adequate markup labels: Many labels used in markup languages such as XML are not meaningful to end users, only to system developers. In order to manipulate the inputs and results of a system, this tool would enable users to refer to the information the system uses and produces in a way that is meaningful to them.
  • Helping users formulate input constraints: Many descriptions of software and Web sites using markup languages such as XML are missing details in terms of the constraints that need to be taken into account when invoking that system. This tool helps users create more precise descriptions of the systems they are using, for example by showing them randomly generated examples of markup data so they can detect and formulate constraints that are missing from the original description.
  • helping users describe a system in terms of the tasks it performs
  • Helping users formulate correct compositions of individual systems
